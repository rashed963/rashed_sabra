# Project Goal (Personal Website / Blog)

## Vision
Build a **simple, minimal, effective, and impactful** personal website where posts are published **first** on the blog (as the official archive) and then shared on LinkedIn.

## Primary objective (This quarter)
- Establish a working end-to-end workflow to publish posts consistently:
  - Write post → add as Markdown file → commit/push → automatic deploy → share link on LinkedIn.

## Audience
- Primary: professionals who follow your writing on LinkedIn and want a canonical archive.
- Secondary: peers and potential collaborators interested in AI/automation/product/technology leadership themes.

## Publishing model
- Blog-first: publish on the personal blog, then share the blog link on LinkedIn.
- Multiple topics/series can run in parallel.

## Technical approach (Agreed)
- **Astro** website (static-first).
- Content stored as **Markdown** (MDX only if needed later).
- **Content Collections** with a stable schema for maintainability.
- GitHub repository as source-of-truth.
- Vercel deploys automatically on push.
- Domain will be added later (should not change the publishing workflow).

## Priorities (Order)
1. Publishing workflow must be frictionless and reliable.
2. Minimal, clean UI that supports reading.
3. Clear information architecture for blog + future series/topics.
4. Easy future extension (tags/series pages, RSS, search) without rewriting core structure.
5. Domain connection (later).

## Tone & style (Site + posts)
- Minimal, clear, practical, and precise.
- Content-first (readability over visuals).
- Avoid unnecessary complexity.

## Out of scope (This quarter)
- Server/VPS setup and maintenance.
- Database-backed CMS.
- Complex design systems.
- Heavy interactive features (search, comments, newsletter) unless proven necessary after stable publishing.
